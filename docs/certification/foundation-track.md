# Foundation Track Certification

> **Track:** Foundation (Phases 1-2)
> **Credential:** AI Analyst — Foundation

---

## Overview

The Foundation Track validates your **AI literacy and workflow design** competencies. You understand how AI works, can write effective prompts, and can identify opportunities to apply AI to business processes.

---

## Required Deliverables

### 1. Prompt Library (Phase 1)

A curated collection of reusable, tested prompts for your professional domain.

**Required Elements:**

| Element | Description |
|---------|-------------|
| **10+ Prompts** | Minimum 10 unique, tested prompts |
| **Categorization** | Organized by use case or task type |
| **CLEAR Framework** | Each prompt demonstrates Context, Load, Expectations, Audience, Restrictions |
| **Testing Notes** | Evidence of iteration and refinement |
| **Use Case Context** | When and why to use each prompt |

**Rubric:**

| Criteria | 1 (Beginning) | 2 (Developing) | 3 (Proficient) | 4 (Exemplary) |
|----------|---------------|----------------|----------------|---------------|
| **Prompt Quantity & Variety** | <10 prompts, limited variety | 10 prompts, some repetition | 10+ prompts, good variety | 15+ prompts, comprehensive coverage |
| **CLEAR Framework Application** | Framework not evident | Partial application | Consistent application | Masterful application with variations |
| **Professional Relevance** | Generic prompts, not domain-specific | Some domain relevance | Clearly relevant to stated profession | Deeply tailored, would impress peers |
| **Testing Evidence** | No testing notes | Minimal testing notes | Clear iteration history | Detailed A/B testing with results |
| **Organization & Usability** | Disorganized, hard to navigate | Basic organization | Well-organized, easy to find prompts | Intuitive system with search/tags |

**Minimum to pass:** 3.0 average

---

### 2. Workflow Automation Proposal (Phase 2)

A detailed proposal for automating a business workflow with AI.

**Required Elements:**

| Element | Description |
|---------|-------------|
| **Current State Analysis** | Documentation of existing workflow |
| **Opportunity Identification** | Which tasks are AI-suitable and why |
| **Proposed Solution** | Detailed AI-enhanced workflow design |
| **Quality Gates** | How errors will be caught and handled |
| **Human-AI Handoffs** | Clear boundaries between human and AI work |
| **Implementation Plan** | Phased approach with milestones |
| **ROI Estimate** | Basic cost-benefit analysis |

**Rubric:**

| Criteria | 1 (Beginning) | 2 (Developing) | 3 (Proficient) | 4 (Exemplary) |
|----------|---------------|----------------|----------------|---------------|
| **Current State Documentation** | Vague or missing | Basic process map | Detailed with pain points | Quantified with metrics |
| **AI Opportunity Analysis** | Generic AI claims | Some task analysis | Clear suitability criteria | Framework-based with risk assessment |
| **Solution Design** | Unclear or unrealistic | Basic design | Detailed and feasible | Innovative and well-justified |
| **Quality & Error Handling** | Not addressed | Mentioned but vague | Clear quality gates | Comprehensive with fallbacks |
| **Human-AI Boundaries** | Unclear | Basic handoff points | Well-defined boundaries | Optimized for human judgment |
| **Implementation Feasibility** | No plan | Basic timeline | Phased with dependencies | Detailed with risk mitigation |
| **ROI Reasoning** | Missing or unrealistic | Basic estimates | Reasonable projections | Well-sourced with assumptions |

**Minimum to pass:** 3.0 average

## Self-Assessment Checklist

Before submitting, verify:

### Prompt Library

- [ ] Contains at least 10 distinct prompts
- [ ] Each prompt uses CLEAR framework
- [ ] Prompts are categorized logically
- [ ] Testing notes show iteration
- [ ] Domain relevance is clear
- [ ] A colleague could use these without explanation

### Workflow Proposal

- [ ] Current workflow is clearly documented
- [ ] AI opportunities are justified (not assumed)
- [ ] Solution is specific and actionable
- [ ] Quality gates prevent errors from propagating
- [ ] Human judgment points are identified
- [ ] Implementation is phased and realistic
- [ ] ROI makes business sense

## Example Deliverables

### Prompt Library Example Structure

```
PROMPT LIBRARY: Marketing Manager

CATEGORY: Content Creation
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Prompt: Blog Post Outline Generator
Version: 3.2 (tested 15+ times)

CONTEXT:
You are an experienced B2B content strategist. You understand
that blog posts must balance SEO value with genuine usefulness.

TASK:
Create a detailed outline for a blog post about [TOPIC].

EXPECTATIONS:
- 5-7 main sections
- 2-3 subsections each
- One key takeaway per section
- Suggested word count per section
- Internal/external link opportunities

AUDIENCE:
[PERSONA] who is researching [PROBLEM].

RESTRICTIONS:
- No clickbait or sensationalized language
- No claims without supporting points
- Avoid jargon; use accessible language

TESTING NOTES:
v1: Too generic, added industry context requirement
v2: Outlines too long, added word count targets
v3: Added link opportunities after SEO feedback
v3.2: Current version, consistently good results

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

### Workflow Proposal Example Structure

```
WORKFLOW AUTOMATION PROPOSAL
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Executive Summary
- Problem: Customer onboarding takes 5 days average
- Opportunity: Reduce to 2 days with AI assistance
- Investment: $25K implementation, $5K/year ongoing
- ROI: $180K annual savings in staff time

1. Current State Analysis
   1.1 Process map (with time per step)
   1.2 Pain points (staff interviews)
   1.3 Error rates and rework
   1.4 Current costs

2. AI Opportunity Assessment
   2.1 Task-by-task suitability scoring
   2.2 Risk assessment per task

3. Proposed Solution
   3.1 AI-enhanced workflow diagram
   3.2 Technology requirements
   3.3 Integration points

4. Quality & Governance
   4.1 Quality gates
   4.2 Human review triggers
   4.3 Error handling procedures

5. Implementation Plan
   5.1 Phase 1: Pilot (4 weeks)
   5.2 Phase 2: Scale (8 weeks)
   5.3 Phase 3: Optimize (ongoing)

6. Investment & Returns
   6.1 Cost breakdown
   6.2 Benefit quantification
   6.3 Payback timeline

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```
